# 개발 로드맵

각 Step은 설명, 구현, 검증, Git 커밋과 푸시까지 완료해야 종료됩니다.

## Step 1. 프로젝트 설계

- 게임의 핵심 루프와 MVP 범위 정의
- 서버 권위 원칙과 기술 구성 정의
- 개발 단계를 문서화

완료 조건: 새로운 참여자가 문서만 읽고 프로젝트의 목적, 범위, 다음 작업을 설명할 수 있다.

## Step 2. 서버 기반 구성

- ASP.NET Core 솔루션 생성
- API, Domain, Infrastructure, Tests 프로젝트 구성
- Health Check와 Swagger 설정

완료 조건: 서버가 실행되고 자동 테스트와 Health Check가 통과한다.

상태: 완료

## Step 3. PostgreSQL 연동

- Docker Compose로 로컬 PostgreSQL 구성
- EF Core DbContext와 첫 Migration 작성
- 사용자 게임 상태 모델 및 제약조건 구현

완료 조건: 빈 DB를 자동으로 구성하고 상태를 저장·조회하는 통합 테스트가 통과한다.

## Step 4. 게스트 인증

- 게스트 계정 생성
- 인증 토큰 발급과 검증
- 사용자별 데이터 접근 제한

완료 조건: 인증되지 않은 요청과 다른 사용자의 데이터 접근이 차단된다.

## Step 5. 방치 보상

- 서버 시간 기반 생산량 계산
- 최대 누적 시간 적용
- 트랜잭션과 중복 지급 방지

완료 조건: 정상, 경계 시간, 중복 요청, 동시 요청 테스트가 통과한다.

## Step 6. 영웅 성장

- 강화 비용 계산
- 골드 차감과 레벨 증가
- 잔액 부족 및 동시 요청 처리

완료 조건: 재화가 음수가 되거나 한 번의 골드로 강화가 중복되지 않는다.

## Step 7. 스테이지 진행

- 전투력과 요구 전투력 계산
- 성공·실패 결과와 스테이지 해금
- 생산 보너스 반영

완료 조건: 동일한 상태와 요청은 항상 동일한 결과를 만든다.

## Step 8. 포트폴리오 마감

- API 오류 계약과 OpenAPI 정리
- 로깅, 예외 처리, 테스트 범위 보강
- 실행 방법, 아키텍처 결정, 데모 시나리오 문서화
- 배포 환경 구성

완료 조건: 제3자가 README만 보고 프로젝트를 실행하고 핵심 시나리오를 검증할 수 있다.

## Step 9. Unity 클라이언트

서버 MVP 완료 후 별도 세션에서 진행합니다.

- 로그인과 서버 연결
- 게임 상태 표시
- 보상 수령, 강화, 스테이지 도전 UI
- 오류 및 네트워크 재시도 처리
